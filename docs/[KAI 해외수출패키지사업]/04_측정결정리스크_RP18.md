# 측정 결정 리스크 (MDR) — RP-18 핵심 & CaliBoard 정렬

> RP-1(교정"주기")의 짝. RP-18은 교정"판정 리스크". 사업기획서 False Accept ≤2% / TUR 4:1 / 가드밴드의 근거.
> 근거: NCSLI RP-18 (1판 2014, 213쪽) 정독. 저자 Howard Castrup(RP-1과 동일 계보).
> 작성일: 2026-06-16.

---

## 0. 한 문장

> **RP-1 = "언제 교정하나(주기)" / RP-18 = "판정이 맞나(리스크)".** 둘 다 소급성 계층(NIST→AFPSL→PMEL→TMDE) 위에서 작동. 저자 동일(Castrup), 참여진에 SpaceX 계측 매니저.

---

## 1. MDR과 두 리스크

**MDR (Measurement Decision Risk)** = 합격/불합격 판정이 틀릴 위험. 두 종류:
| 리스크 | 영문 | 무엇 | 결과 |
|---|---|---|---|
| **오수락** | **FAR** (False Accept Risk) | 불합격품을 합격으로 오판 | 위험 — 불량 장비 현장 투입 |
| **오기각** | **FRR** (False Reject Risk) | 합격품을 불합격으로 오판 | 비용 — 불필요 재교정·수리 |

- 사업기획서 "**False Accept ≤2%**" = 이 **FAR**.
- MDR은 소급성 계층 **각 단계마다 누적**됨 (RP-18 §Role of MDR). 임채욱 선임 DCC 소급성 사슬과 동일 그림.

## 2. ★★ "4:1 TUR"에 대한 RP-18의 입장 — 발표 필수 (§3.5)

사업기획서·Z540.3는 "TUR ≥ 4:1"을 군계약 필수로 내세움. **그런데 RP-18은 4:1을 "조잡한 도구"라고 비판**:
- **§3.5.2**: "4:1은 [허용오차÷확장불확도] 비율일 뿐, **리스크를 특정 수준으로 통제하지 못한다. 기껏해야 조잡한(crude) 도구.**"
- "리스크를 쉽게 계산할 수 있는 오늘날 왜 아직 이걸 쓰는지 이해하기 어렵다."
- **§3.6 권고**: "4:1은 UUT 사전 규격내확률을 **전혀 모를 때만** 써라. 정보 있으면 **고전적/베이지안 분석**. **베이지안이 더 낫다.**"
- TUR 정의(§3.5.1): TUR = (허용오차 범위) ÷ (2×U95). 양쪽·대칭 허용오차에만 유효. 단측엔 부적용.

### 발표 표현 안전성
| 표현 | 안전성 |
|---|:---:|
| "TUR 4:1 충족" | ✅ (최소 요건. 사실) |
| "TUR 4:1이 리스크를 통제/보장한다" | ⚠️ **위험 — RP-18이 "통제 못 함" 명시** |
| "4:1을 넘어 **FAR을 정량 계산**한다" | ✅✅ **최강 차별화 — RP-18 권장 상위 방법** |

> 발표 메시지: "경쟁사는 단순 4:1. 우리는 RP-18이 권장하는 **리스크 정량화(FAR % 계산)**로 간다." = 표준 근거 있는 한 수 위 포지셔닝.

## 3. 가드밴드 (§5.3) — CaliBoard Phase G-3 정렬

- **가드밴드 = 합격선(A)을 규격선(L) 안쪽으로 좁힘** (승수 g<1). 효과: **FAR↓ FRR↑** (트레이드오프).
- 종류: Test Guardband(시험 합격선) / Reporting Guardband(성적서 보고선).
- **리스크 기반 가드밴드(§B.4.2)**: "목표 FAR(예 2%)을 정하고 → 가드밴드 폭을 **역산**."

### CaliBoard 격차
- 현재 Phase G-3(step3): **ILAC-G8 4단계 판정**(conformant/conditional-pass/conditional-fail/non-conformant). 가드밴드 "개념"은 있음.
- RP-18: **목표 FAR에서 가드밴드 폭을 역산** = 리스크 기반 설계.
- → 개선하면 "False Accept ≤2% 보장"이 **진실**이 됨 (사업기획서 KPI 직결). 현재는 판정만, FAR % 역산 없음.

## 4. 역할 분담 (완성)
```
교정주기 (언제?)   → RP-1   (measurement reliability, 85% target, MLE, 데이터 20~40건)
교정판정 (맞나?)   → RP-18  (FAR/FRR, 가드밴드, 4:1 비판→리스크 정량화/베이지안)
                  둘 다 저자 Castrup, 소급성 계층 위에서 작동
```

| 발표 주장 | 근거 | 보유 |
|---|---|:---:|
| 데이터 기반 교정주기 | RP-1 + ISO 10012 + TO §3.4 | ✅ |
| False Accept ≤2% / FAR·FRR | RP-18 | ✅ |
| 가드밴드 리스크 통제 | RP-18 §5.3 | ✅ |
| "4:1 넘어 리스크 정량화" 차별화 | RP-18 §3.6 | ✅ |

## 5. 발표 주의사항 (거짓 방지)
1. "TUR 4:1이 리스크 보장" ❌ → RP-18이 "조잡, 통제못함" 명시. "4:1은 최소요건, 우리는 정량화"로.
2. "False Accept ≤2% 보장" — 현재 코드 미달(가드밴드 판정만, FAR% 역산 없음). 개선 후 진실.
3. 베이지안 MDR = RP-18 최선. 고급 주장 시 언급하면 권위↑ (통계 전문성 필요).
