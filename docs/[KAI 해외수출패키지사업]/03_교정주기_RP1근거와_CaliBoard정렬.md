# 교정주기 — RP-1 국제표준 근거 & CaliBoard 정렬 분석

> 우리 사업·ICMPM 발표의 **기술적 심장**. "데이터로 교정주기를 바꿀 수 있는가"의 법적·기술적 정당성.
> 근거: NCSLI RP-1(2010/2019) 정독 + TO 00-20-14 §3.4 + ISO 10012 §7.1.2 + ISO/IEC 17025 §6.4.7 + CaliBoard 코드 조사.
> 작성일: 2026-06-16.

---

## 0. 한 문장 결론

> **표준(ISO 10012·17025)은 "데이터로 교정주기를 조정하라"고 *허용·권장*한다. 미 공군 AFMETCAL은 이를 신뢰성 85% 목표로 *실현*한다. 그 구체적 방법론이 NCSLI RP-1이고, RP-1의 최상위 방법(MLE)은 대규모 데이터를 요구한다 → 데이터 가진 자(KTL)만 할 수 있다.**

---

## 1. 두 체계 대조 — KOLAS vs 미 공군, 누가 주기를 바꿀 수 있나

**핵심: 둘 다 "변경 허용"이 맞다. 차이는 "입증 부담"이다.** (사용자 인상 "KOLAS에선 못 바꾼다"는 표준이 아니라 관행)

| | 미 공군 (TO 00-20-14) | KOLAS (ISO 17025/10012) |
|---|---|---|
| 주기 변경 가능? | ✅ 명문 (§3.4 "modified") | ✅ 명문 (§6.4.7 / §7.1.2) |
| 변경 근거 | 모집단 통계 데이터 | 교정이력·SPC 데이터 |
| 변경 권한 | AFMETCAL 본부 (중앙 집중) | 각 기관이 자체 문서화 절차로 |
| 실무 체감 | 본부가 데이터로 일괄 조정 → 현장은 따름 | **기관이 입증책임 짊어짐 → 부담 커서 권장주기 그대로 쓰는 보수적 관행** |

**원문 근거**:
- **ISO 10012 §7.1.2**: "확인 주기를 결정·변경하는 방법은 문서화 절차에 기술되어야 한다. 주기는 필요시 검토·조정되어야 한다. [지침] **교정이력 데이터와 SPC 기록**이 주기 변경 판단에 사용될 수 있다." (참고문헌 [17] OIML D10)
- **ISO/IEC 17025 §6.4.7**: "교정 프로그램을 수립하여야 하며, 교정 상태 신뢰성 유지를 위해 검토·조정되어야 한다."
- **TO 00-20-14 §3.4.5**: "데이터를 축적한 활동은 변경 데이터·권고안을 AFMETCAL에 제출 → AFMETCAL 승인으로 변경."

→ **결론: 표준은 길을 열어뒀다. 막은 건 "입증할 데이터·방법이 없음". 그 입증을 자동화하는 게 우리.**

## 2. NCSLI RP-1 = 방법론의 실체 (표준이 비워둔 "How")

- 제목: *Recommended Practice-1: Establishment and Adjustment of Calibration Intervals* (NCSL International, 4판 2010 / 2019 재발행, 174쪽)
- AF TO·ISO가 "통계로 도출하라"고만 하고 위임한 **실제 알고리즘이 여기 다 있음**.
- 집필: Fluke·Agilent·Boeing·美해군(NSWC)·DOE Pantex 등 = 사실상 전 세계 교정주기 방법론 교과서.

### RP-1의 5가지 방법 (데이터량↑ = 정확도↑)
| 방법 | 데이터 요구 | 정확도 | 비고 |
|---|---|:---:|---|
| General Interval | 없음 | 최하 | 모든 장비 단일주기. 신규품 초기값 |
| Borrowed Intervals | 외부 | 하 | 제조사·타기관 주기 차용 |
| Engineering Analysis | 설계지식 | 중 | 유사장비·제조사권장·회로분석(OOTR) |
| Reactive (A1/A2/A3) | 소량 | 중상 | 결과 반응 조정. **A1·A2 비권장, A3만 권장** |
| **MLE (S1/S2/S3)** | **20~40건** | **최상** | 최대우도추정. 신뢰성모델 피팅 |

### 핵심 개념 정의
- **측정신뢰성(Measurement Reliability)** = "그 시점에 장비가 규격 이내(in-tolerance)일 확률" (§2.2). 시간↑ → 불확도 성장 → 신뢰성↓.
- **신뢰성 목표(Reliability Target)** = 조직이 정한 최소 허용 신뢰성. **AF의 85%가 이것** (§2.3). 보편상수 아니라 미공군의 선택값.
- **최적주기(Optimal Interval)** = 신뢰성 목표 충족 + 비용효율 + 반응성 + 부산물 (§2.5). 신뢰성 곡선이 목표(85%)와 만나는 시점.
- **불확도 성장 산출법** = 공식계산 ❌ / **데이터에 신뢰성 모델 곡선 피팅** ⭕ (§3.6, Appendix D에 8개 모델). 그래서 데이터 필요.
- **MLE = Maximum Likelihood Estimation (최대우도추정)**: 관측 데이터를 가장 잘 설명하는 신뢰성 곡선 파라미터를 역추정. RP-1 신뢰성 피팅의 수학 엔진.

### ★ MLE "20~40건"의 정확한 의미 — 묶음(grouping) 단위
- 개별 장비 1대의 이력 20건이 **아님**. RP-1 §2.7.1: "단일 시리얼로는 충분한 데이터가 현실적으로 거의 안 모임."
- 묶음 단위: **단일 시리얼 ~20건 / 모델번호 ~30건 / 계측기 클래스 ~40건**.
- **Fleet 범위** = "불확도 성장이 비슷할 것으로 기대되는 동질 그룹" = 기본 **같은 제조사·모델**. 데이터 부족 시 클래스(기능군)로 넓히되 정밀도 희생. "그냥 모든 토크렌치"는 ❌ (모델 다르면 드리프트 다름).

## 3. ★ CaliBoard 현재 위치 판정 + 개선 경로

CaliBoard 교정주기 분석(Phase G)은 RP-1 기준 **혼합(hybrid), 중하위 방법**:

| CaliBoard 단계 | RP-1 분류 | 평가 |
|---|---|---|
| step1 Baseline (제조사권장/k-tools/12개월) | Borrowed + General | ✅ 일치 |
| step2 Trend Drift (사용률·가속·추세 규칙) | **Reactive A1** | ⚠️ RP-1이 "비권장"으로 명시 |
| step3 Guard Band (불확도/ILAC-G8) | §3.4 Guardband | ✅ 개념 일치 |
| step5 Crossing (선형회귀 외삽) | 독자 방식 | △ RP-1 5방법에 없음 |
| step4 Peer (유사장비, **미반영**) | Similar Items | ✅ 단 미반영이라 가치 못 살림 |

**핵심 파일**: `web/src/lib/cycle-analysis.ts`(1483줄, 5단계), `cycle-analysis-dummy.ts`(회귀·crossing·peer), `components/equipment-detail/tab-cycle-analysis.tsx`(UI), `api/ai/cycle-analysis/route.ts`(LLM 평가).

### 두 가지 격차(gap)
- **격차① 측정신뢰성 개념 부재 (치명적)**: 코드에 "신뢰성 85% 목표"도, 신뢰성 곡선도 없음. `confidence`(데이터충분도)는 있으나 `reliability`(규격내 확률)와 다른 개념. crossing은 "오차가 tolerance에 닿는 시점"이지 "신뢰성이 85%로 떨어지는 시점"이 아님. = 개별 외삽, 모집단 신뢰성 아님.
- **격차② Reactive A1**: step2가 RP-1 비권장 방법 A1 구조. RP-1 권장은 A3 또는 MLE(S2).

### 3대 개선 경로 (우선순위)
1. **측정신뢰성 축 도입** (최우선): crossing에 더해 "이 주기에서 모집단 몇 %가 in-tolerance인가" 계산. → "85% 목표로 산출" 문구가 진실이 됨.
2. **Peer를 모집단 데이터로 승격**: 현재 미반영 step4를 신뢰성 추정 데이터 소스로. k-tools 9311건을 모델/클래스로 묶으면 MLE 데이터 요건 충족. = 사업기획서 "유사장비 빅데이터 예측"과 일치.
3. **A1 → A3/MLE 격상** (장기, 통계 전문성 필요. RP-1 §1.2 경고: "설계명세서 아님, 통계·공학 전문가 참여 필수").

## 4. ★★ ICMPM 발표 핵심 메시지 3종 세트 + 문구 헛점 점검

### 발표 서사 (Before → 표준의 정도 → KTL 해자)
1. **현재(Before, 정직한 출발점)**: "우리는 **관행상** KOLAS 권장주기에 따라 **fixed-term(고정주기)** 교정을 한다. 안전하지만 과잉교정(비용↑)/과소교정(리스크↑) 가능성."
2. **표준의 정도(正道)**: "그런데 ISO 10012 §7.1.2 / 17025 §6.4.7은 fixed term을 강제하지 않는다. 오히려 데이터 기반 조정을 **허용**한다. 미 공군 AFMETCAL은 이를 **신뢰성 85% 목표**로 실현한다."
3. **KTL 해자**: "데이터 기반 최적화(RP-1 MLE)는 모델/클래스당 20~40건을 요구한다. 개별 기관은 못 채운다(§2.7.1). **대규모 교정 모집단을 가진 KTL만 가능.** = 표준이 만들어준 진입장벽."

> 한 줄: **"교정주기 최적화는 알고리즘이 아니라 데이터가 만든다. 그리고 데이터는 KTL의 것이다."**

### 문구 헛점 점검 — 현재 코드 기준 안전/위험
| 문구 | 현재 가능? | 조건 |
|---|:---:|---|
| "데이터 기반 (ISO 10012/17025 **허용**)" | ✅ 진실 | 없음 |
| "AFMETCAL·RP-1 **원리에 기반**(based on principles)" | ✅ OK | 약한 표현 유지 |
| "RP-1 **방법론 구현**" | ❌ | 개선3 (A3/S2) 필요 |
| "**85% 신뢰성 목표로 산출**" | ❌ | 개선1 (신뢰성 축) 필요 |

**안전 문구(현재도 진실)**: "CaliBoard 교정주기 분석은 ISO 10012(§7.1.2)·ISO/IEC 17025(§6.4.7)가 허용하는 데이터 기반 접근을 따르며, 미 공군 AFMETCAL(TO 00-20-14)과 NCSL RP-1이 제시하는 '불확도 성장 통제' 원리에 기반한다."

**주의사항 3가지**:
- ① "85% 목표로 산출"은 개선1 전엔 **거짓** (코드에 신뢰성 곡선 없음).
- ② "85%는 미 공군 값" — 한국/KOLAS target이 85%란 근거 없음. "AFMETCAL이 채택한 85%를 **참조**한다"로 표현.
- ③ "KOLAS가 금지한다" ❌ → "**관행상** fixed term을 따른다"로. (표준은 허용)

### 발표 전 확인 필요 (반박 방지)
- KTL이 실제로 모델/클래스별 20~40건을 보유/접근 가능한가? (KTL 교정이력 규모, k-tools 9311건의 모델별 분포 확인) → 사실 뒷받침되면 "데이터 해자" 메시지 반박 불가.

## 5. 교정주기 관련 문서 최종 점검 (발표 정확성 — "거짓말 방지")

**RP-1이 교정주기 "방법론(How)"의 정점이 맞다.** 다이어그램·RP-1 교차점검 결과:
- 교정주기 "어떻게 산출/조정하나"를 다루는 유일한 방법론서 = RP-1. 나머지는 결과값(33K·CMS 주기표) 또는 요구(ISO·TO "하라").
- → "데이터로 주기 바꾼다" 주장엔 RP-1로 충분. 더 볼 것 없음.

**단, RP-1이 스스로 위임하는 형제 문서 (발표 주제에 따라 필요)**:
- **RP-18 (Estimation and Evaluation of Measurement Decision Risk)** — RP-1이 본문 5회 위임.
  **TUR / False Accept ≤2% / 가드밴드 = "판정 리스크"의 근거는 RP-1이 아니라 RP-18.**
  가드밴드·False Accept를 발표에서 강조하면 RP-18 입수 필수 (RP-1로 우기면 거짓).
- RP-12 (Determining and Reporting Measurement Uncertainty) — 불확도(GUM) 산출 상세. 불확도 깊게 갈 때만.
- RP-3 (Calibration Procedures) / RP-5 (MTE Specifications) — 보조, 발표 비필수.

**축 구분 (헷갈리면 거짓 발언)**: RP-1=교정"주기" / RP-18=교정"판정 리스크(false accept/reject)". 별개의 두 축.
