## 0. 한 장으로 보는 전체 논리 (Closed-Loop Reliability)

```
[현재 / Before]
  KOLAS 권장주기 기반 fixed-term 교정 (관행상). 과잉/과소교정 가능.
        │ 표준은 사실 다른 길을 허용한다 ↓
[표준의 정도(正道)]
  ISO 10012 §7.1.2 / 17025 §6.4.7 = "교정이력 데이터로 주기 조정하라" (허용·권장)
  미 공군 AFMETCAL TO 00-20-14 §3.4 = 신뢰성 85% 목표로 데이터 기반 주기 운영 (실현 사례)
        │ 그 구체적 방법론 = ↓
[방법론]
  RP-1 (교정주기) : MLE로 신뢰성 곡선 피팅 → 85% 닿는 시점 = 최적주기. 단 데이터 20~40건 필요.
  RP-18 (판정리스크) : FAR/FRR, 가드밴드. "4:1은 조잡, 리스크 정량화/베이지안이 낫다."
        │ 데이터 20~40건을 어떻게 채우나? + 주기 늘린 공백은? ↓
[해결 = 키오스크 중간점검]   ★ 닫힌 루프의 핵심
  중간점검(ISO 17025 §6.4.10 + RP-1 SPC + RP-18 FAR저감)
   → ① 데이터 대량 생성 (MLE 데이터 요건 충족)
   → ② 교정 사이 공백 모니터링 (FAR 통제)
   → MLE 권장주기 예측에 환류 → 키오스크 장치 개발 검토
        │ 누가 이걸 할 수 있나? ↓
[KTL 해자]
  MLE는 대규모 데이터 요구 → 데이터 가진 자만 가능 → KTL의 구조적 강점.
```

## 1. 키오스크 중간점검 흐름 (★발표 핵심 서사)

```
① 중간점검 실시      ← ISO 17025 §6.4.10(정식 요구) + RP-1 §3(SPC 보충) + RP-18(FAR 저감)
        ↓ 데이터 생성
② 데이터 축적        ← 정식교정(연1회) + 중간점검(수시) = 데이터 밀도 ↑↑
        ↓ 데이터 충분
③ MLE 권장주기 예측   ← RP-1 (MLE 데이터 20~40건 요건을 중간점검이 채움)
        ↓ 실현 수단
④ 키오스크 장치 개발  ← 중간점검을 자동·상시화하는 HW (Future Work → 사업 정당성 핵심)
```

**키오스크가 일석이조**:
- 문제 A (RP-1): "MLE 데이터 20~40건 부족" → 키오스크가 데이터 대량 생성으로 해결
- 문제 B (RP-18): "주기 연장 시 공백 FAR 리스크" → 키오스크가 공백 모니터링으로 통제

**주의**: 중간점검 ≠ 교정. 정식 교정을 대체하지 않음(소급성·불확도 요건 다름). "늘린 주기를 안전하게 뒷받침"이 정확. ISO 17025 §6.4.10은 "필요한 경우" 실시(전 장비 의무 아님).

## 2. 표준끼리 입장차 — Z540.3 vs RP-18 (발표 차별화 포인트)

**같은 미국(NCSL 계열) 표준인데 입장이 갈린다 — 충돌이 아니라 위계·시점 차이:**
- **Z540.3 (2006)**: "TUR ≥ 4:1 충족하라" = 요구사항(계약 합격선)
- **RP-18 (2014)**: "4:1은 조잡한 도구, 리스크 통제 못 함. 직접 계산하라. 베이지안이 낫다" = 권장 관행
- 이유: ① 4:1은 컴퓨팅 없던 시절(50~80s) 근사치, 지금은 계산 가능 ② 요구(법) vs 권장(모범사례) 층위 차이

**무기화**: 경쟁사는 "4:1 충족" 수준. 우리는 "RP-18 권장 = 리스크 정량화/베이지안"으로 한 세대 앞섬.
**주의**: "Z540.3가 틀렸다" ❌ → "4:1 충족하면서 RP-18 권장 수준까지 나아간다" ⭕ (4:1은 여전히 유효 계약요건).

## 3. 발표 문구 안전성 종합 (거짓말 방지 체크리스트)

| 문구 | 안전성 | 조건 |
|---|:---:|---|
| 현재 KOLAS 권장주기 fixed-term (관행상) | ✅ | "관행상"으로. "KOLAS가 금지"는 ❌ |
| ISO 10012/17025가 데이터 기반 조정 허용 | ✅ | 원문 확인됨 (§7.1.2, §6.4.7) |
| AFMETCAL 85% 신뢰성 목표 **참조** | ✅ | "참조". "우리 기준이 85%"는 ❌(미공군값) |
| AFMETCAL·RP-1 **원리에 기반** | ✅ | 약한 표현 |
| 중간점검 = ISO 17025 §6.4.10 | ✅ | 원문 확인. 단 "교정 대체"는 ❌ |
| TUR 4:1 충족 | ✅ | 최소 요건 |
| 데이터 해자 (MLE 20~40건) | ✅ | KTL 데이터 규모 사실 확인 후 |
| RP-1 **방법론 구현** | ❌→ | 개선3(A3/S2) 후 진실 |
| **85% 목표로 산출** | ❌→ | 개선1(신뢰성축) 후 진실 |
| **False Accept ≤2% 보장** | ❌→ | 리스크기반 가드밴드 도입 후 진실 |
| TUR 4:1이 리스크 보장 | ❌ | RP-18이 "통제 못함" 명시 |

→ **현재 코드로 진실인 안전 문구로 발표하고, 개선 1·리스크가드밴드 구현하면 강한 문구로 격상.**

## 6. ★ 청중별 프레이밍 분기 (KAI vs ICMPM) — 자료 이원화 원칙

**같은 분석(닫힌 루프 논리)을, 청중에 따라 완전히 다른 옷으로.** 이 문서들(00~05)은 "분석 원천"이고, 두 산출물로 갈라진다.

| | KAI 분석/제안용 | ICMPM 발표용 |
|---|---|---|
| 위치 | `[KAI 해외수출패키지사업]/` (이 폴더) | `docs/2026-06-24_ICMPM2026/발표자료/` |
| 청중 | KAI 경영진·DAPA (한국, 친미 방산) | **CSSC 704연구소 (중국 국유 조선·방산), 국제학회** |
| 전면 표준 | **AFMETCAL·TO 00-20-14** 전면 (NATO 수출 강점) | **ISO/IEC 17025·ISO 10012·BIPM·ILAC** 전면 |
| AFMETCAL 취급 | 핵심 모델 | "선진 사례 중 하나"로 축소 or 생략 |
| 소급성 표현 | NIST→AFPSL→PMEL | **BIPM→각국 NMI(KRISS·NIM 등)** 중립 |
| 성격 | 사업 수주·기능명세·매출 | 학술·기술 보편성·중립 |

**왜 분기 필수 (중국 청중)**: CSSC=中國船舶集團(국유 최대 조선·방산). 미·중 패권 감안, "미 공군 표준 따른다"는 ① 자국표준 무시 ② 미국 종속 인상 위험.
**해법**: 우리 논리의 진짜 뿌리는 **국제표준(ISO 10012/17025, GUM/VIM-BIPM)** + **미국 민간학회(NCSL RP-1/RP-18, 군 아님)**. AFMETCAL은 "구현 사례 하나"일 뿐 빼도 논리 안 무너짐.
**중국 우호 요소**: ISO 17025 기반 → 중국 CNAS 인정체계와 상호 호환(ILAC MRA). 소급성도 BIPM 아래 NIST·KRISS·NIM 동등하게.
**주의**: "Z540.3/AFMETCAL이 틀렸다" ❌. 그냥 국제표준을 전면에 세우고 미국 군표준은 뒤로.

→ ICMPM 발표 전용 논리 문서는 `2026-06-24_ICMPM2026/발표자료/`에 별도 작성 (이 분석을 국제표준 프레이밍으로 재구성).
