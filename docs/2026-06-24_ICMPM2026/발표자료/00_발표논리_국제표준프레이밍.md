## 0. 프레이밍 대원칙 (KAI 자료와 정반대)

**같은 기술 논리, 다른 옷.** KAI 제안서는 AFMETCAL 전면이지만, 여기선 절대 금지.

| 쓸 것 ✅ | 쓰지 말 것 ❌ |
|---|---|
| ISO/IEC 17025, ISO 10012 | 미 공군 AFMETCAL, TO 00-20-14 (전면 배제) |
| GUM(JCGM 100), VIM (BIPM 국제) | "미군 85% 기준" |
| NCSL RP-1, RP-18 ("국제 계측 학회 모범사례"로) | NIST 단독 강조 |
| BIPM → 각국 NMI(KRISS·NIM 등) 중립 소급성 | NIST→AFPSL→PMEL (미군 계층) |
| ILAC/CNAS/KOLAS 상호인정 | NATO·FA-50·KF-21 (방산 수출 색채) |

> NCSL RP-1/RP-18은 **미국 민간 학회**(군 아님) 문서라 인용 가능. 단 "미국"보다 "국제 계측학계 모범사례"로 표현.

---

## 1. 발표 핵심 메시지 (한 문장)

> **"측정 신뢰성과 교정주기 최적화는 ISO/IEC 17025와 국제 계량학(metrology)의 보편 원리다.
> 이를 *데이터로 실현*하는 것이 핵심이며, 그 데이터 역량이 KTL의 강점이다."**

---

## 2. 발표 논리 흐름 (국제표준 버전 닫힌 루프)

```
[문제] 현재 산업계는 권장주기 기반 fixed-term 교정. 과잉/과소교정 가능.
   │
[국제표준의 정도] ISO 10012 §7.1.2 / ISO 17025 §6.4.7
   = "교정이력 데이터로 주기를 검토·조정하라" (국제표준이 이미 허용·권장)
   │
[방법론] 국제 계측학계 모범사례 (NCSL RP-1: 측정신뢰성 모델·MLE / RP-18: 결정리스크 FAR·FRR)
   = 데이터에 신뢰성 곡선 피팅 → 목표 신뢰성 닿는 시점 = 최적주기
   │
[데이터 공백 해결 = 중간점검] ISO/IEC 17025 §6.4.10 "intermediate check"
   → 키오스크 중간점검이 ① MLE 데이터 생성 ② 주기연장 공백 리스크(FAR) 통제
   → 중간점검 데이터를 교정주기 모델로 환류 = 닫힌 루프(Closed-Loop) 신뢰성 관리
   │
[실현] KTL의 대규모 교정 데이터 = MLE 실현 역량 (데이터 가진 자만 가능)
   + 키오스크 장치 개발 검토
```

---

## 3. 중국 청중 우호 요소 (적극 활용)

- **상호인정(MRA)**: ISO 17025 기반이라 중국 **CNAS** 인정체계와 호환 (ILAC MRA). "국경 넘는 측정 신뢰성".
- **중립 소급성**: SI 단위(BIPM, Sèvres) 아래 NIST·KRISS·**NIM(중국계량과학연구원)**이 동등. 특정국 우위 아님.
- **보편성 강조**: "이건 한 나라의 방식이 아니라, 측정과학의 보편 언어다."
- DCC(디지털 교정성적서) = 국제적으로 진행 중인 흐름(BIPM·산업4.0)과 연결 → 정치색 없음.

---

## 4. 발표 정직성 체크 (거짓말 방지 — KAI 자료와 동일하게 적용)

- "ISO 10012/17025가 데이터기반 조정 **허용**한다" → ✅ 진실
- "중간점검은 ISO 17025 §6.4.10" → ✅ 진실. 단 "교정 대체 아님" 명시
- "우리는 RP-1 MLE로 주기 산출한다" → ❌ 현재 코드 미구현. "RP-1 **원리에 기반**한 데이터 분석" 으로
- "신뢰성 목표로 산출" / "FAR ≤2% 보장" → ❌ 개선 전엔 못 씀
- KTL 데이터 해자 → KTL 실제 데이터 규모 확인 후 주장
