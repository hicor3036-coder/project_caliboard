### s24 — Tier 4 · Reactive (정석, 계단차트) ★이번 세션 전면 재작성
**화면 (확정):**
- 제목 "STEP 2 · NCSLI RP-1 — Tier 4 · Reactive: just react to each result"
- 부제 "pass, lengthen; fail, shorten. It **reacts, not predicts**"
- 좌측 = StaircaseChart (Pass→+10% / Fail→−45%, 계단: 6→6.6→7.3→8→8.8→9.7→(fail)5.3→2.9→"? no fixed answer", "jumps around — no forecast")
- 우측 메인:
  > **Tier 4 — the first to use real data.** But just this one wrench.
  > ✓ **Pass** → lengthen the interval (+10%)
  > ✗ **Fail** → shorten it (−45%)   ※"the RP-1 §B.2 value at Rt:85%" 같은 출처표기는 화면에서 제거(궁금증 안 만들기)
  > It just **reacts** to the last result — it never looks ahead.
  > Now we finally used data — but only **one wrench**, and it just swings.
  > One wrench can't decide. → Ask the **whole fleet · Tier 5**
- 하단 별도 박스 = 질문 방어용 공식: "RP-1 §B.2: b = 1 − (1−a)^(Rt/(1−Rt)) / a = step up on pass / b = step down on fail · Rt = reliability target"
- ★"not recommended"는 화면에 안 박음(차트 캡처 안 잔존 텍스트는 재캡처 필요·우선순위 낮음). 멘트로도 말 안 함 — 장점/단점만.
**스크립트:**
- "Tier 4 is the first method that uses real data — this wrench's own pass and fail history."
- "The rule is simple. If the last calibration passed, we make the interval a little longer. If it failed, we make it much shorter."
- "How much longer or shorter — that follows RP-1, Appendix B point two."
- "The good thing is, it's simple — you can use it right away, with just this one wrench's data."
- "But look at the chart — it goes up, then one fail drops it hard. It only reacts to the last result, so it can't really follow the long-term trend."
- "One wrench isn't enough to decide. To really predict the interval, we ask the whole fleet — Tier 5."

### s25 — Tier 5 · MLE (클라이맥스) ★이번 세션 페어링 재정리
**화면 (확정):**
- 제목 "STEP 2 · NCSLI RP-1 — Tier 5 · MLE: the fleet decides"
- 부제 "hundreds of the same wrench, together → the best interval is ~8.4 months"
- 좌측 = MLE 신뢰성곡선(harsh subgroup 174 units, 100 N·m=8.4mo 결정포인트, 85% target, λ=0.019/mo, n=404). 85% target 출처=DAFMAN 21-113 §2.6.14.2 화면 유지.
- 우측 메인 (s24 헤드라인 스타일로 페어링):
  > **Tier 5 — the whole fleet decides.** Look at hundreds of the same wrench, used the same way.
  > Each year we know if each one passed or failed. As time passes, fewer stay within spec — so we can see how reliability drops over time. It stays above 85% up to **8.4 months**. That's the best interval.
  > One wrench can fool you. But, Hundreds give a steady, trustworthy answer.
- 하단 결론박스 (★사다리 회수 + 데이터 기반 선언):
  > **The more data, the better the answer.**
  > Tiers 1–3 — no real data, just experience and specs.
  > Tier 4 — real data, but from one wrench alone.
  > Tier 5 — data from the whole fleet.
  > → One wrench tells you little; the fleet tells you the truth.
  > That's **data-driven calibration** — calibration isn't a certificate, **it's data**.
**스크립트:**

- "It stays above our 85% target up to about 8.4 months. So 8.4 months is the best interval." (★85%=referenced·널리 쓰는 목표로. "follow DAFMAN" 표현 안 씀)

- "Tier 4 — real data, but from one wrench alone."

### s26 — STEP 3 · Safeguard (본론·발표 내용 마지막) ★이번 세션 전면 재작성
**화면 (확정):**
- 제목 "STEP 3 · Safeguard — watching the instrument between calibrations"
- 부제 "Required by RP-1 & ISO/IEC 17025 — and even more important once the interval is dynamic"
- 좌측 = 키오스크 이미지(Intelligent Verification Kiosk). 우측 = drift 차트(점검점 빨간 빗금 + "Frequent kiosk checks (rough, but often)" + drift 지점 원).
- 하단 본문 (사용자 메시지 4박자):
  > When we set the interval by data, we also need to **watch the instrument between calibrations** — this is what **RP-1 and ISO/IEC 17025 ask for**.
  > ① It keeps the measurements trustworthy in daily use.
  > ② It **catches early drift** — before **reliability breaks down**.
  > ③ A **full calibration is slow and costly**, so we're building an easy, **kiosk-style check: quick and frequent**.
  > It doesn't replace calibration — **it watches between them.** · ISO/IEC 17025 §6.4.10, §7.7
- ★초점 = "조기 발견 → 조치(수리/교정)". 주기 늘림/당김 논쟁은 화면에서 뺌. (차트 "8.4→7 pulled in"은 ②early drift의 결과로 위상에 부합 = 당김✅, 늘림은 MLE만)
**스크립트 (발표 내용 마지막 — 다음 장은 Thank you):**
- "So the data let us extend the interval. But is a longer interval really safe? That's the last step."
- "RP-1 and ISO 17025 both ask us to check the instrument between calibrations. And once the interval is dynamic, that becomes even more important."
- "A quick check, often, keeps the measurements trustworthy in daily use."
- "And it catches early drift — before reliability breaks down."
- "But to catch it early, you have to check often — and a full calibration is too slow and costly to do that."
- "So we're developing a kiosk — for a quick daily check, right on the floor."
- "It's not as accurate as a full calibration — but it's fast and easy, so you can check the instrument anytime."
- "If something looks wrong, you catch it early — and act before any bad measurement goes out: repair it, or send it in for calibration."
- "It's not calibrate every week. It doesn't replace calibration — it just watches between them."
- **[섹션3 마무리]** "So this is the way forward. The data tells us the best interval. And a simple check makes sure it stays safe."
- **[발표 전체 마무리 — 한 박자 쉬고]** "So that's our story today — AI checks the data, DCC turns it into real data, and data sets the best interval. Thank you for listening."

- **RP-1 티어**: General §4.3 / Borrowed §4.4 / Engineering §4.5 / Reactive §4.6 / MLE §4.7
- **RP-1 Borrowed §2.6.2**: "intervals determined by an **external organization**" (예: 미 DoD/GIDEP/Navy MIDAS). maker 아님 → "external recommendation" 정답.
